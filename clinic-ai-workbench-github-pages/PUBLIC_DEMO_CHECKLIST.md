# 公开演示发布检查清单

## 已处理

- 使用患者A、患者B、患者C、患者D、患者E等脱敏演示别名。
- 演示患者联系方式为空或仅允许录入脱敏尾号。
- 演示照片只保存说明，不内置真实影像。
- 登录页和系统内顶部加入“公开演示版”提示。
- 页面加入 `noindex, nofollow`，降低被搜索引擎收录概率。
- 加入 `.nojekyll`，避免 GitHub Pages 对静态文件做 Jekyll 处理。
- 发布目录只包含静态演示所需文件和说明文档。

## 发布前再次确认

- 不上传真实患者姓名、手机号、身份证、住址、人脸照片。
- 不上传真实处方、真实收费、真实医生工号或内部经营数据。
- 不上传浏览器缓存、截图缓存、测试脚本输出或本机日志。
- 不把前端权限描述成真实安全体系。
- 对外说明这是原型演示，不是可上线生产系统。

## 对外介绍建议

可以这样介绍：

“这是白癜风专科门诊系统的公开演示版，用于展示导医分诊、电子病历、专科评估、处方监管、经营质控和权限管理等流程。演示数据均已脱敏，系统仅用于需求沟通和产品原型展示。”

不要这样介绍：

“这是可以直接给医院上线使用的系统。”

## 后续上线生产需要补齐

- 后端数据库和文件存储
- 真实账号密码登录
- 服务端权限校验
- 不可篡改审计日志
- 病历签名、归档、修订的合规流程
- 图片/影像受控存储和访问授权
- 收费、退款、药品、库存、医保或第三方接口
- 备份、灾备、日志留存和安全运维方案
