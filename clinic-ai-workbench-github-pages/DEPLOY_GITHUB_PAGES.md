# GitHub Pages 部署步骤

## 方式一：上传到新仓库

1. 在 GitHub 新建仓库，例如 `clinic-ai-workbench-demo`。
2. 将本目录内所有文件上传到仓库根目录。
3. 进入仓库 `Settings`。
4. 打开 `Pages`。
5. 在 `Build and deployment` 里选择 `Deploy from a branch`。
6. 分支选择 `main`，目录选择 `/root`。
7. 保存后等待 GitHub Pages 构建完成。
8. 访问 GitHub 给出的 Pages 地址。

## 方式二：作为已有仓库的子目录

如果放在已有仓库子目录，例如 `/clinic-ai-workbench/`，请把本目录所有文件放入该子目录，并将 GitHub Pages 发布目录指向对应分支和目录。

当前代码使用相对路径引用资源，适配仓库根目录和子目录部署。

## 发布前检查

- 首页能打开。
- 登录页有“公开演示版”提示。
- 可以选择 `管理员001` 登录。
- 左侧菜单、患者工作台、权限管理能正常切换。
- 刷新页面后仍能进入系统。
- 没有真实患者资料、真实照片、真实手机号或真实病历。

## 常见问题

如果页面空白，优先检查：

- 是否漏传 `app.js` 或 `analysis.js`。
- 是否在 GitHub Pages 设置里选错了发布目录。
- 浏览器控制台是否提示文件路径 404。

如果修改演示数据后想恢复：

- 在系统右侧患者栏点击“恢复演示数据”。
- 或清理浏览器当前站点的本地存储后刷新页面。
