# 白癜风门诊系统公开演示版

这是一个用于需求沟通和流程展示的白癜风专科门诊系统前端演示版。系统覆盖导医挂号分诊、患者工作台、电子病历、照片时间轴、白癜风专科评估、项目/疗程、处方审核、院长看板、账号角色权限和审计日志等原型功能。

## 重要提示

- 本项目是公开演示版，不是生产医疗系统。
- 禁止录入真实患者姓名、手机号、身份证、人脸照片、病历、处方或收费数据。
- 演示数据保存在访问者自己的浏览器本地，不会上传服务器。
- 前端账号和权限只用于原型演示，不能替代真实后端鉴权、服务端权限校验和不可篡改审计。
- AI/规则分析只用于演示风险分层和沟通草稿，不替代医生诊疗判断。

## 演示入口

打开 `index.html`，或部署到 GitHub Pages 后访问站点首页。

默认演示账号在登录页下拉选择：

- 管理员001：可查看导医台、患者工作台、院长看板、权限管理、审计日志
- 医生001 / 医生002：可查看归属患者、病历、处方和专科评估
- 导医001：可使用导医台挂号分诊
- 治疗师001：可查看治疗执行相关记录
- 回访001：可查看回访任务
- 院长001：可查看经营和医疗质控看板

## 适合展示的内容

- 导医台：查重、建档、挂号、分诊、候诊队列
- 患者工作台：患者检索、基本信息、病历、检查、照片、治疗、回访
- 白癜风专科能力：部位图、VASI/BSA、照片对比、光疗剂量、疗效趋势
- 经营与质控：项目/疗程、处方审核、院长看板、审计日志
- 管理后台：账号管理、角色管理、科室字典、菜单和操作权限

## 发布文件

GitHub Pages 需要保留这些文件：

- `index.html`
- `styles.css`
- `app.js`
- `analysis.js`
- `sample-patients.csv`
- `.nojekyll`
- `README.md`
- `DEPLOY_GITHUB_PAGES.md`
- `PUBLIC_DEMO_CHECKLIST.md`

不要把本机工作目录里的 `tests`、`scripts`、`work`、截图缓存、浏览器缓存、真实资料或其他临时文件上传到公开仓库。
